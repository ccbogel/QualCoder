## Zwei Codes systematisch vergleichen

### Ziel
Zwei inhaltlich nahe oder potenziell überlappende Codes so vergleichen, dass ihr jeweiliges analytisches Profil klarer wird, saubere Fälle, Grenzfälle und vermutlich falsch codierte Fälle sichtbar werden, Ein- und Ausschlusskriterien für zukünftiges Codieren präzisiert werden.

1. Materialbasis prüfen

Für beide Codes jeweils ansehen:
- Codename
- Codememo
- die codierten Segmente

Wenn möglich:
- zuerst eine breite Auswahl aus verschiedenen Interviews lesen
- bei sehr wenigen Segmenten transparent machen, dass die Einschätzung vorläufig ist

2. Analytische Leitfrage formulieren
- Für jeden der beiden Codes kurz festhalten:
- Was genau wird mit diesem Code analytisch sichtbar?
- Welche Perspektive auf das Material eröffnet er?
- Worin unterscheidet er sich vom Vergleichscode?

3. Segmente systematisch klassifizieren
Alle gelesenen Segmente eines Codes in drei Gruppen einordnen:

a) Sauber
Das Segment passt klar und ohne größere Zweifel zum Code.
Kriterium:
die im Memo oder Codenamen angelegte Bedeutung ist explizit oder sehr deutlich implizit im Material vorhanden

b) Grenzfall
Das Segment passt teilweise, aber auch ein anderer Code wäre plausibel.
Kriterium:
mehrere Lesarten sind möglich
der Code trifft etwas Relevantes, aber nicht unbedingt den primären Sinn der Passage

c) Vermutlich umcodieren / kritisch prüfen
Das Segment passt nach derzeitigem Stand eher nicht gut zum Code.
Kriterium:
der zentrale Gehalt der Passage liegt klar woanders
die bisherige Codierung wirkt zu breit, unscharf oder irreführend

4. Für jedes Segment knapp begründen
Zu jedem eingeordneten Segment kurz notieren:
- Warum ist es ein sauberer Fall / Grenzfall / fraglicher Fall?
- Was ist das zentrale Merkmal der Passage?
- Welche Alternative wäre plausibel?
Die Begründung soll knapp sein, aber immer am Material bleiben.

5. Vergleichende Auswertung schreiben
Am Ende die beiden Codes direkt gegeneinander schärfen:
- Was leistet Code A, was Code B nicht leistet?
- Wo überlappen sie?
- Wo besteht Verwechslungsgefahr?
- Welcher Code ist aktuell enger/schärfer, welcher breiter/unscharfer?
- Welche Blindstellen entstehen, wenn man nur mit einem der beiden Codes arbeitet?

7. Methodische Regeln
- Nicht vorschnell umcodieren, sondern erst analytisch prüfen
- Doppelkodierung zulassen, wenn zwei Aspekte wirklich gleichzeitig zentral sind
- nicht nur auf einzelne Wörter reagieren, sondern auf den Sinnzusammenhang zwischen 
  - empirischer Aussage,
  - Interpretation,
  - und Codierentscheidung
  klar unterscheiden
- bei schmaler Datenbasis ausdrücklich sagen, dass die Einschätzung vorläufig ist
